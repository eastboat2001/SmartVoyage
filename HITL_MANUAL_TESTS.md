# HITL_MANUAL_TESTS

本文档用于手工验证 LangGraph HITL 人工审核链路。默认自动化 LangSmith 基础集不再覆盖这些场景。手工回归既可在 CLI 中完成，也可在 FastAPI Web 页面中完成。

## 前置条件

- 已启动后端服务：`.\.venv\Scripts\python.exe run_all.py`
- 已启动命令行：`.\.venv\Scripts\python.exe main.py`
- 或已启动 Web 页面：`.\.venv\Scripts\python.exe run_all.py --with-web` 后访问 `http://127.0.0.1:8501`
- 当前演示用户使用 `demo_user`
- 如需固定相对日期，可在 `.env` 中设置：`SMARTVOYAGE_NOW_OVERRIDE=2026-03-21T09:00:00+08:00`

## 测试目标

- 下单在审批前会暂停并要求 `yes/no`
- 退票在审批前会暂停并要求 `yes/no`
- 改签在审批前会暂停并要求 `yes/no`
- `transport_decision` 自动下单场景会先给建议，再进入审批，再执行下单
- 多轮补参 + HITL 可以正常衔接

## 手工测试用例

### 1. 直接下单审批

输入：

```text
当前用户：demo_user
帮我预订2026-03-21北京到上海的高铁票，二等座1张
```

期望：

- 系统先返回审批提示，不直接下单
- 提示里包含 `G5`、`二等座` 或等价票务摘要

继续输入：

```text
yes
```

期望：

- 返回预订成功
- 回复中包含订单号
- 再执行 `查询我的订单` 时能看到该订单

### 2. 下单审批拒绝

输入：

```text
当前用户：demo_user
帮我预订2026-03-21北京到上海的高铁票，二等座1张
```

期望：

- 系统先进入审批态

继续输入：

```text
no
```

期望：

- 返回未执行实际下单
- `查询我的订单` 不应出现新订单

### 3. 退票审批

先准备一张已订高铁票，然后输入：

```text
当前用户：demo_user
帮我退掉2026-03-21北京到上海的高铁票
```

期望：

- 系统先进入退票审批态

继续输入：

```text
yes
```

期望：

- 返回退票成功
- 订单状态变为已取消

### 4. 改签审批

先准备一张 `2026-03-21 北京 -> 上海 G5 二等座` 已订订单，然后输入：

```text
当前用户：demo_user
把我2026-03-21北京到上海的高铁票改签到2026-03-22二等座
```

期望：

- 系统先进入改签审批态

继续输入：

```text
yes
```

期望：

- 返回改签成功
- 新订单应为 `2026-03-22` 的 `G13` 二等座

### 5. 多轮退票补参 + 审批

第一轮输入：

```text
当前用户：demo_user
帮我退票
```

期望：

- 系统追问缺失信息

第二轮输入：

```text
2026-03-21北京到上海的高铁票
```

期望：

- 系统进入审批态

第三轮输入：

```text
yes
```

期望：

- 返回退票成功

### 6. 多轮改签补参 + 审批

第一轮输入：

```text
当前用户：demo_user
帮我改签高铁票
```

期望：

- 系统追问缺失信息

第二轮输入：

```text
把2026-03-21北京到上海的票改签到2026-03-22二等座
```

期望：

- 系统进入审批态

第三轮输入：

```text
yes
```

期望：

- 返回改签成功

### 7. transport_decision 自动下单 + 审批

输入：

```text
根据2026-03-21上海的天气，帮我判断从北京去上海坐高铁还是飞机更合适，如果有合适票就直接帮我订一张
```

期望：

- 回复先包含：
  - `天气判断`
  - `出行建议`
  - `票务结果`
- 然后进入审批态，而不是直接执行下单

继续输入：

```text
yes
```

期望：

- 最终回复仍保留完整建议上下文
- 同时附带 `预订结果`
- 回复中包含订单号

## 建议记录项

每次手工回归建议记录：

- 输入原文
- 是否进入审批态
- `yes/no` 后的最终回复
- 是否符合预期
- 若失败，对应 `logs/app.log`、`logs/mcp.log` 的关键错误
- 如果是在 Web 页面联调失败，也建议同时检查 `logs/web.log`
